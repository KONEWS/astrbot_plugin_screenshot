📸 截图分析插件 (AstrBot Plugin)这是一个为 AstrBot 框架开发的高稳定屏幕截图与 AI 视觉分析插件。它不仅能解决 Windows 高分屏下的截图缩放裁剪问题，还能直接调用大语言模型（LLM）对屏幕画面进行智能分析。

🌟 核心优势 (优点)突破系统缩放限制：底层调用 Windows 的 Per-Monitor DPI 接口 (SetProcessDpiAwareness(2))，完美解决 Win11/Win10 环境下开启 125%、150% 等高分屏缩放时，截图发虚或只能截取局部画面的痛点。
无缝接入 AI 视觉：截图后自动将图片转换为 Base64 格式直传给 AstrBot 底层的大模型，实现真正的“看图说话”，无论是代码报错审查还是画面内容提取都能轻松应对。
极度自由的指令解析：支持“直接截图”、“延迟截图”、“自定义分析提示词”的任意组合，满足复杂的自动化需求。零资源泄露 (自清洁机制)：内建后台异步清理器。每次截图发送并分析完毕后，会在 30 秒后自动销毁存储在硬盘上的原图；在插件重载或卸载时，也会自动清空所有后台任务，杜绝内存和磁盘空间溢出。

⚠️ 局限性与使用要求 (缺点)为了保持客观，安装前请务必阅读以下局限性：强依赖视觉大模型 (Vision)：本插件的分析功能依赖于你在 AstrBot 中配置的大模型。
你必须使用支持视觉能力（Vision）的模型（例如：GPT-4o、Claude 3.5 Sonnet、Gemini 1.5 Pro 等）。如果配置的是纯文本模型，执行到最后一步时必然会报错。系统强绑定：受限于底层 API，本插件仅支持 Windows 系统。部署在 Linux 云服务器（无头环境）或 macOS 上将会直接拦截并提示不支持。仅限主屏幕：目前版本的截图引擎固定截取系统的主显示器（Monitor 1）画面。如果外接了多块屏幕，它暂时无法截取副屏的内容。物理显示限制：电脑不能处于锁屏、休眠状态。若是使用远程桌面连接，在最小化远程窗口时系统会停止画面渲染，此时大概率会截出黑屏或报错。

🚀 安装与部署自动安装（推荐）：将 main.py、metadata.yaml、requirements.txt 三个文件打包为一个 .zip 压缩包。
进入 AstrBot 的 WebUI 管理面板。在【插件管理】页面选择“上传插件”，选中该 .zip 文件并安装。重启 AstrBot，框架会自动在后台为你安装运行所需的依赖 (mss 和 Pillow)。
